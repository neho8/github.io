Understanding Jenkins

Date: 22.10.2023 + Video below


Automation and Efficiency

Jenkins is a crucial tool for DevOps, serving as an open-source automation server. It automates code building, testing, and deployment, making it easier for developers to create and manage CI/CD pipelines.


What Is CI/CD pipeline?

CI/CD (Continuous Integration/Continuous Delivery) is a process that drives software development through a path of building, testing, and deploying code before providing it to the users. Automating the process minimizes human error and maintains a consistent operation for how software is released.


Installing Jenkins With Docker:

Docker lets us run our own Jenkins Container and host it in our local machine:

1. Install Docker: If you haven't already, install Docker on your system.

2. Run Jenkins Container: Open your terminal where you will pull and run the Jenkins container. For our coming projects, we will run Jenkins using the following commands:

docker network create jenkins

docker run --name jenkins --restart=on-failure --detach --network jenkins --env DOCKER_HOST=tcp://docker:2376 --env DOCKER_CERT_PATH=/certs/client --env DOCKER_TLS_VERIFY=1 --volume jenkins-data:/var/jenkins_home --volume jenkins-docker-certs:/certs/client:ro --publish 8080:8080 --publish 50000:50000 jenkins/jenkins

3. Access Jenkins: Open your web browser and navigate to http://localhost:8080. You will be prompted to enter an initial admin password. Retrieve it by running these commands in the terminal:

docker exec -it jenkins /bin/bash

cat /var/jenkins_home/secrets/initialAdminPassword

4. Set Up Jenkins: Follow the setup wizard to complete the Jenkins installation, create an admin user, and install recommended plugins.


Using the Jenkins User Interface:

The Jenkins user interface is intuitive and provides access to essential features. Here's a quick overview:

Dashboard: The dashboard displays an overview of all your Jenkins jobs, their statuses, and recent builds.

New Items: To create a new job, click on "New Item". Choose a project type (e.g., Freestyle project or Pipeline) and configure your job.

Build Now: Manually trigger a build by clicking the "Build Now" option in your job's menu.

Console Output: After a build, view the console output to see the progress and any errors.


Creating Your First Jenkins Automation Build:

Let's make a simple Jenkins build that echoes "Hello, World"

1. New Item: From the Jenkins dashboard, click "New Item," name your project, and select "Freestyle project".

2. Configure Project: In the project configuration, under the "Build" section, click "Add build step" and select "Execute shell".

3. Command: In the "Command" field, enter the following: echo "Hello, World".

4. Save: Save your project configuration.

5. Build: Click "Build Now" to initiate your first Jenkins automation build.

You'll see the build's progress in the console output, and the output "Hello, World!" will be displayed as everything runs successfully.


Customizing Your Jenkins Automation Build:

While the "Hello, World" example is a straightforward demonstration of Jenkins' capabilities, you can create more complex automation builds tailored to your project's specific requirements. Jenkins offers extensive customization options:

Source Code Management: Jenkins supports integration with various version control systems such as Git. You can configure your automation build to pull source code from a repository, ensuring that the latest code is tested and deployed.

Build Triggers: Jenkins allows you to set up build triggers. You can configure your automation build to run on specific events, like code commits or pull requests. This ensures that your CI/CD pipeline is triggered automatically, reducing the need for manual intervention.

Plugins: Jenkins offers a wide range of plugins to extend its functionality. You can integrate plugins for different purposes, such as code analysis, testing frameworks, deployment to various cloud platforms, and more. These plugins enhance Jenkins' capabilities and allow you to build complex automation pipelines.

Parameterized Builds: Parameterized builds allow you to customize your automation builds by accepting user-defined parameters. This is useful when you want to perform different actions based on user input or deploy to different environments.

Notifications: Jenkins can notify your team about build status and issues. You can set up email notifications, integrate with popular communication tools like Slack, or even use custom scripts to notify relevant team members when a build fails or succeeds.

Environment Setup: Depending on your project requirements, you can configure Jenkins to set up specific environments for your builds. For example, you can define build agents with specific software configurations to ensure consistent and reproducible builds.

Testing and Reporting: Jenkins provides support for running tests and generating detailed reports. You can combine your preferred testing frameworks and reporting plugins to monitor code quality and identify issues early in the development process.


Conclusion

Jenkins is your trusted partner in the realm of DevOps, simplifying CI/CD automation, and offering endless possibilities for automation and customization. Embrace Jenkins, and it will empower your development journey like never before.
